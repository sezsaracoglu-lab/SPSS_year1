SPSS Practical Guide (Set 1)
Core procedures
Frequencies
distributions · how often · percentages · percentiles
Use to see how often each value appears. Great first step to "get to know" your dataset. Use for nominal & ordinal variables. Also use to get percentile values (e.g. what score = 25th percentile).
Analyze → Descriptive Statistics → Frequencies
  1. 1
    Move variable(s) into the Variable(s) box
  2. 2
    Click Statistics → tick Mean, Median, Mode, SD; for percentiles type the number (e.g. 25) → Add → Continue
  3. 3
    Click Charts → Histogram (scale) or Bar Chart (nominal/ordinal) → Continue
  4. 4
    Click OK → read the Statistics table in Output Viewer
💡Cumulative % = "what proportion scored this value or lower". Output: Output Viewer.
"how often""what % of participants""distribution""describe the sample""what value is the Nth percentile"
Case Summaries
list individual cases · show raw scores per participant
Use when you want to list individual cases with their values. Good for checking specific participants or verifying data entry. Does NOT summarise — it shows everything individually.
Analyze → Reports → Case Summaries
  1. 1
    Move the variable(s) you want to display into the Variables box
  2. 2
    Optionally add a grouping variable in Grouping Variable(s)
  3. 3
    Tick "Show case numbers" to identify each row
  4. 4
    Click OK → table shows each case with its values; Total N at the bottom
💡Think of it as a printed class register — every student's mark is shown individually. Output: Output Viewer.
"list all cases""show individual scores""for each participant"
Compare Means
mean and SD per group in one table
Use when you want to compare average scores across groups. You are NOT testing significance yet — just looking at the numbers side by side. Use before deciding which test to run.
Analyze → Compare Means → Means
  1. 1
    Move the outcome variable to Dependent List
  2. 2
    Move the grouping variable to Independent List
  3. 3
    Click Options → confirm Mean, N, SD are in Cell Statistics → Continue
  4. 4
    Click OK → table shows mean, N, and SD for each group
💡More efficient than Split File when you just need group means in one table. Output: Output Viewer.
"mean difference""compare groups""average per group"
Recode into Different Variables
create categories · reverse scores · collapse groups
Use when you need to change the values of an existing variable. Classic use: reversing a negatively worded item (1→5, 2→4…). Also for collapsing age or scale ranges into categories.
Transform → Recode into Different Variables
  1. 1
    Move original variable into Input Variable → Output Variable box
  2. 2
    Type a new name in Output Variable Name → click Change
  3. 3
    Click Old and New Values
  4. 4
    For ranges: click "Range" → e.g. 1 through 10 → New value: 1 → Add. Repeat for all categories.
  5. 5
    For reversing 1–5 scale: old 1→new 5, old 2→new 4, old 3→new 3, etc.
  6. 6
    Click Continue → OK → check Data View for new column
⚠ALWAYS use "Different Variables" — never overwrite the original. Nothing appears in Output Viewer — check Data View only.
"reverse score""negatively worded item""create categories""collapse groups into"
Compute Variable
total scores · mean of items · composite variables
Use when you want to create a new variable using arithmetic on existing variables. Most common: calculating a scale total or mean score from multiple questionnaire items.
Transform → Compute Variable
  1. 1
    Type a name in Target Variable (e.g. anxiety_total)
  2. 2
    In Numeric Expression, type your formula
  3. 3
    Mean: MEAN(item1, item2, item3) — handles missing values automatically
  4. 4
    Sum: SUM(item1, item2, item3)  ·  Manual: (item1 + item2 + item3) / 3
  5. 5
    Click OK → new variable appears in Data View (not Output Viewer!)
💡Use MEAN() not manual division — it handles missing data. Output: Data View only.
"calculate a total score""create a composite""sum the items""scale score"
Count Values Within Cases
count how many times a value appears per person
Use when you want to count how many times a specific value appears across multiple variables FOR EACH CASE. Classic use: counting how many symptoms a participant endorsed.
Transform → Count Values Within Cases
  1. 1
    Type a name in Target Variable
  2. 2
    Move all the variables to count across into the Variables box
  3. 3
    Click Define Values → specify the value to count (e.g. value = 1) → Add → Continue
  4. 4
    Click OK → new variable in Data View shows the count per row
💡Counts per ROW (per person), not per column (per variable). Output: Data View only.
"how many symptoms""number of items endorsed""count occurrences per person"
Split File
run any analysis separately per group
Use when you want to run ANY analysis separately for different groups. Like putting a divider in a filing cabinet — everything after gets sorted into separate folders.
Data → Split File
  1. 1
    Select "Organize output by groups"
  2. 2
    Move grouping variable (e.g. gender) to Groups Based on
  3. 3
    Click OK → status bar shows Split File On
  4. 4
    Run your analysis → output repeats for each group separately
  5. 5
    TURN OFF: Data → Split File → "Analyze all cases" → OK
⚠Forgetting to turn Split File off is the #1 SPSS exam mistake. Every subsequent analysis stays split.
"separately for each group""for males and females separately"
Z-scores (save standardised)
z-score for a specific participant · standardise · outlier detection
Use when you want to standardise scores so different variables are on the same scale. Z-scores tell you how many SDs a score is above/below the mean. Also to identify outliers (|Z| > 3.29).
Analyze → Descriptive Statistics → Descriptives
  1. 1
    Move variable(s) into the Variable(s) box
  2. 2
    Tick "Save standardized values as variables" checkbox at the bottom
  3. 3
    Click OK → new column Z[varname] appears in Data View
  4. 4
    Find the requested participant ID row → read their Z value
  5. 5
    To show more decimals: Variable View → Z variable → change Decimals to 5
💡Z = 0 → score equals the mean. Z = +1 → one SD above mean. |Z| > 3.29 → extreme outlier. Output: Output Viewer + Data View (Z column).
"standardise""z-score""how many SDs above/below mean""outlier"
SPSS Guide Set 2
Additional procedures
Explore
normality check · Shapiro-Wilk · full descriptives with plots
Essential BEFORE running a t-test or ANOVA — you need to check normality. Use to examine outliers, skewness, kurtosis, and the shape of your distribution.
Analyze → Descriptive Statistics → Explore
  1. 1
    Move your continuous variable(s) to the Dependent List box
  2. 2
    Optionally move a grouping variable to the Factor List to get stats per group
  3. 3
    Click Plots → tick "Normality plots with tests" → tick Histogram
  4. 4
    Under Display: select "Both" (Statistics + Plots)
  5. 5
    Click Continue → OK
💡Output: Descriptives table (mean, SD, skewness, kurtosis), percentiles, Shapiro-Wilk (use for N < 50), histogram, Q-Q plot, boxplot. Sig. > .05 = data IS normal (keep parametric). Sig. < .05 = NOT normal.
"check normality""test assumptions""before running t-test""is the distribution normal""outliers"
Rank Cases
which percentile does a score belong to · rank order
Use when you want to rank participants on a variable, or to find which percentile a specific score belongs to. Converts raw scores into rank positions or percentile values.
Transform → Rank Cases
  1. 1
    Move variable into the Variable(s) box
  2. 2
    Under "Assign Rank 1 to": choose Smallest value (ascending) or Largest value (descending)
  3. 3
    Click Rank Types → tick Fractional rank as % (for percentile questions) → Continue
  4. 4
    Click OK → new variable P[varname] appears in Data View
  5. 5
    Find the requested participant ID → read their value → round as required
💡Tied scores get the average rank (e.g. 4th and 5th tied → both get 4.5). Output: Data View only.
"rank the participants""which percentile does this score belong to""rank order"
Select Cases
filter to a subgroup · compound conditions · count N
Use when you need to filter to a specific subgroup, especially when two conditions must be true simultaneously (something Split File can't do).
Data → Select Cases
  1. 1
    Select "If condition is satisfied" → click If...
  2. 2
    Type your condition: e.g. edu = 2 & remedial = 0 or geo > 5 & hist > 5
  3. 3
    Continue → OK → unselected rows show a diagonal slash in Data View
  4. 4
    Run your analysis → only selected cases are included
  5. 5
    TURN OFF: Data → Select Cases → "All cases" → OK
💡To count cases meeting criteria: after selecting, run Frequencies and read Valid N. The & operator means BOTH conditions must be true.
Crosstabs
two categorical variables · percentages within groups · chi-square
Use when examining the relationship between two categorical variables. Shows how the frequency of one variable is distributed across categories of another. Also used for the Chi-Square test of independence.
Analyze → Descriptive Statistics → Crosstabs
  1. 1
    Move one variable to Row(s), the other to Column(s)
  2. 2
    Click Statistics → tick Chi-Square if significance test is needed
  3. 3
    Click Cells → tick Observed counts + Row %, Column %, and/or Total % as needed
  4. 4
    Click Continue → OK
💡Row % = "of all people in this row, what % are in each column?" Chi-Square assumption: all expected cell counts ≥ 5. Output: Output Viewer.
"contingency table""cross-tabulation""chi-square""is there an association"
Independent Samples T-test
test mean difference between two separate groups
Use when comparing means of two SEPARATE (independent) groups on one continuous variable. Groups are independent — different people in each group (males vs females, control vs treatment).
Analyze → Compare Means → Independent-Samples T Test
  1. 1
    Move the continuous outcome variable into Test Variable(s)
  2. 2
    Move the grouping variable into Grouping Variable
  3. 3
    Click Define Groups → enter the two group codes (e.g. 1 and 2) → Continue
  4. 4
    Click OK → check Levene's Test Sig. first
  5. 5
    Sig. > .05 → TOP row (equal variances assumed) · Sig. < .05 → BOTTOM row → read t value
⚠NEVER skip Levene's test. It determines which row to read. Report t, not F. t can be negative — include the sign.
"do two groups differ""compare males and females""independent groups""between-subjects"
Paired Samples T-test
same participants measured twice · before/after · pre-post
Use when comparing means from the SAME participants measured at two time points or conditions. Data is paired — each person provides two scores. Key distinction: same people = paired; different people = independent.
Analyze → Compare Means → Paired-Samples T Test
  1. 1
    Click the first variable (e.g. pre-test score) → then Ctrl+click the second variable (e.g. post-test)
  2. 2
    Click the arrow → both variables move as a pair into Paired Variables
  3. 3
    Click OK
💡Output: Paired Samples Statistics (mean, N, SD) · Paired Samples Correlations · Paired Samples Test table (mean difference, SD of diff, 95% CI, t, df, Sig. 2-tailed). If Sig. < .05 → significant change.
"before and after""pre-test post-test""same participants twice""within-subjects""did scores change"

10
Test the difference on the Dutch language test between Dutch and non-Dutch children. What is the t-score? (3 dp)
Dataset 4
Independent T-test
▼
  1. 1
    Analyze → Compare Means → Independent Samples T-test
  2. 2
    Move Dutch language test score into Test Variable(s)
  3. 3

Pre-analysis checklist, the 5 most common mistakes, output location table, and keyword lookup — everything you need in the exam room.

Pre-analysis checklist
Recode or Compute? Check Data View for the new variable column — not Output Viewer
Running a t-test? Run Explore FIRST to check normality (Shapiro-Wilk)
Independent t-test? Always check Levene's before reading the t value
Used Split File or Select Cases? Turn it OFF before the next analysis
Crosstabs with Chi-Square? Check all expected cell counts are ≥ 5
The 5 most common mistakes
Forgetting to turn Split File off. Every analysis stays split until you go Data → Split File → Analyze all cases. Say so in your exam answer.
Recoding into the Same Variable. Always "Different Variables". Recoding same permanently destroys original data.
Looking in Output Viewer for Recode / Compute / Rank Cases. These only appear in Data View.
Mixing up Q8 vs Q9 direction. "What score = 25th percentile?" → Frequencies. "What percentile = this score?" → Rank Cases.
Ignoring Levene's test. Sig. > .05 → top row. Sig. < .05 → bottom row. Report t, not F.
